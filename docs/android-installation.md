# Установка NDT Ninja на Android-планшет

## Способ 1: PWA (рекомендуется)

### Через Chrome для Android

1. **Откройте игру в браузере**
   - Перейдите на сайт с игрой через Chrome
   - Убедитесь, что игра работает корректно

2. **Установите как приложение**
   - Нажмите кнопку меню Chrome (⋮) в правом верхнем углу
   - Выберите «Установить приложение» или «Добавить на главный экран»
   - Подтвердите установку

3. **Запуск**
   - Иконка появится на главном экране и в ядре приложений
   - Запуск в полноэкранном режиме без адресной строки

### Преимущества PWA
- ✅ Работает офлайн (после первого запуска)
- ✅ Автоматические обновления
- ✅ Не занимает место как APK
- ✅ Плавная интеграция с системой

## Способ 2: Развёртывание на локальном хостинге

### Подготовка

```bash
# Сборка проекта
npm run build

# Проверка сборки
npx serve dist -p 8080
```

### Доступ с планшета

1. Запустите сервер на ПК в той же сети Wi-Fi
2. Узнайте IP-адрес ПК: `ip addr show` (Linux) или `ipconfig` (Windows)
3. На планшете откройте: `http://<IP_ПК>:8080`
4. Установите как PWA (см. выше)

### Пример: `npx serve dist -l 8080`

```bash
cd /home/user/projects/games/ninja
npm run build
npx serve dist -l 8080 --cors
```

Флаг `--cors` нужен для доступа с других устройств.

## Способ 3: GitHub Pages (онлайн)

1. Закоммитьте изменения:
   ```bash
   git add .
   git commit -m "feat: добавить PWA поддержку"
   git push origin main
   ```

2. Включите GitHub Pages в настройках репозитория:
   - Settings → Pages → Source: Deploy from a branch
   - Branch: main/master, Folder: root

3. Откройте URL вида `https://username.github.io/repo-name/` на планшете
4. Установите как PWA

## Touch-управление на планшете

Игра уже поддерживает touch:
- **Свайп пальцем** — разрез фигур
- **Pointer Events** — единая модель для мыши/тача/стилуса
- **Блокировка gesture** — нет случайного скролла/refresh

### Настройки для оптимальной работы

В Chrome на Android:
1. `chrome://flags/#touch-action-none`
2. Убедитесь, что включена оптимизация мультитач

## Диагностика проблем

### Игра не устанавливается как PWA
- Проверьте наличие `/manifest.json` в корне
- Откройте DevTools → Application → Manifest

### Офлайн-режим не работает
- Проверьте Service Worker в DevTools
- Очистите кэш и перезагрузите страницу

### Плохая производительность на планшете
- Переключите рендерер в `src/config/game.ts`:
  ```ts
  type: Phaser.AUTO, // вместо Phaser.CANVAS
  ```
- Соберите заново: `npm run build`

## Технические детали

### Структура PWA

```
dist/
├── index.html              # Главная страница + registration SW
├── manifest.webmanifest    # Mанифест приложения
├── sw.js                   # Service Worker (кэширование)
├── workbox-*.js            # Workbox runtime
└── assets/
    ├── index-*.js          # Бандл игры (Phaser + код)
    └── ...
```

### Кэшируемые ресурсы

- JavaScript бандлы
- CSS стили
- Изображения (фон, иконки)
- Звуковые эффекты

### Offline-first стратегия

После первого визита все ресурсы кэшируются. При отсутствии сети игра загружается из кэша Service Worker.

## Команды разработки

```bash
# Локальная разработка с PWA
npm run dev

# Сборка для продакшена
npm run build

# Предпросмотр сборки
npm run preview

# Тестирование на устройстве (нужен local-ip)
npm run build && npx serve dist -l 8080 --cors
```

## Следующие шаги

Для более нативного опыта можно использовать:
- **Capacitor** — обёртка вокруг PWA для Google Play
- **TWA (Trusted Web Activity)** — публикация PWA в Google Play как native app
- **Bubblewrap** — создание TWA манифеста
