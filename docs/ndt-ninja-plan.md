# 🥷 NDT-Ninja: план реализации

> Пошаговый план реализации клона Fruit Ninja в теме **неразрушающего контроля (NDT)**.
>
> Архитектура, выбор стека и описание систем — в отдельном документе:
> [`ndt-ninja-architecture.md`](./ndt-ninja-architecture.md). Кратко: стек
> **Phaser 3 + Matter.js + PolyK**, платформа десктоп + мобильный, реалистичная физика твёрдых тел.

**Принцип плана.** Каждый этап имеет **verifiable checkpoint** — объективный критерий успеха,
проверяемый в браузере, а не «чтоб работало». Этап можно считать завершённым только когда
checkpoint пройден.

---

## Пошаговый план (фазы 0–7)

| # | Этап | Задачи | Checkpoint (критерий успеха) |
|---|---|---|---|
| **0** | **Скаффолд проекта** | Vite + TypeScript + Phaser 3; пустая сцена; FIT-scale; `touch-action: none` на канвасе; dev-сервер | ✅ Канвас рендерится на десктопе и мобиле; свайп по странице не скроллит её |
| **1** | **InputSystem** | Pointer Events (`pointerdown/move/up/cancel`); буфер траектории свайпа; `getCoalescedEvents()` для быстрых движений; отрисовка следа | ✅ Рисуется плавный след за пальцем/мышью без разрывов на быстрых свайпах |
| **2** | **SpawnDirector + физика** | Конфиг `physics: { default: 'matter' }`; спавн NDT-объекта как `Matter body` (`fromVertices`) + спрайт; баллистическая дуга + гравитация + вращение | ✅ Болт взлетает дугой, вращается в полёте, падает под гравитацией |
| **3** | **Ядро слайсинга** | SliceSystem (пересечение segment свайпа ∩ полигон тела); BodySplitter через **PolyK** → 2 новых тела + 2 спрайта с маской; импульс разлёта вдоль нормали реза | ✅ Свайп через тело → 2 физических осколка разлетаются в разные стороны |
| **4** | **Игровой контент** | Болты, гайки, линейки, эталоны (режутся); **трубы-бомбы** (взрыв при разрезе); спавн по волнам | ✅ Все объекты режутся; труба при разрезе = game over + взрыв + screen-shake |
| **5** | **SwordSystem + FX** | 4 меча (кованый / сварка / плазма / радиация); след меча, частицы, шейдеры glow; переключение мечей | ✅ Каждый меч даёт свой визуальный след/частицы; переключение работает и влияет на рез |
| **6** | **ScoreSystem + прогрессия** | Очки за разрез; множитель комбо за серию в окне времени; волны с растущей сложностью; рекорд в `localStorage` | ✅ Счёт растёт, комбо-множитель применяется, волны усложняются, рекорд сохраняется между сессиями |
| **7** | **Полировка + перф** | Object pooling осколков/частиц; texture atlas; аудит draw calls; аудио + haptics (мобайл); тюн Physical Resolution и субшагов физики | ✅ Стабильные 60fps на среднем мобиле, draw calls <50; звук и виброотклик работают |

## Зависимости между фазами

```
0 (скаффолд)
 └─ 1 (ввод)
     └─ 2 (физика/спавн)
         └─ 3 (слайсинг)  ← ключевая техническая развилка
             ├─ 4 (контент + бомбы)
             └─ 5 (мечи + FX)
                 └─ 6 (счёт + прогрессия)
                     └─ 7 (полировка + перф)
```

Фазы 4 и 5 можно вести **параллельно** после прохождения фазы 3 (ядра слайсинга).

## Риски и открытые вопросы

> Это вопросы, которые нужно снять **до или в ходе** соответствующих фаз. Технические обоснования —
> в [`ndt-ninja-architecture.md`](./ndt-ninja-architecture.md).

1. **Туннелирование осколков в Matter.js** (нет CCD) — быстрые осколки проходят сквозь другие тела.
   Снимать на **фазе 3/7**: субшаги физики (`runner.isFixed`, увеличенный `delta`) или raycast-ворота
   на линии реза. Если не поможет — кандидат на переход на Planck.js/Rapier (есть CCD).
2. **Сравнительная производительность физдвижков** при десятках/сотнях осколков **не верифицирована**
   (verify-агенты упали на rate-limit). Снимать **до фазы 7** целевым бенчмарком Matter.js vs Rapier
   на целевых устройствах — это определит, нужно ли менять стек.
3. **Themed-эффекты мечей** (шейдеры glow для плазмы/радиации, частицы сварки) — без источников,
   требуют прототипирования на **фазе 5**. Запасать время на эксперименты с Phaser-частицами и
   WebGL-фильтрами.
4. **Дизайн геймплея NDT** — продуктовые решения, не исследованные: game over vs штраф за трубу,
   кривая сложности волн, бонус-механика «радиация подсвечивает скрытые дефекты». Снять **до фазы 6**
   через игровые тесты прототипа.
5. **Вогнутые фрагменты при слайсинге** — Matter.js работает только с выпуклыми телами. Если рез
   даёт вогнутые половинки, нужен `poly-decomp`. Проверить на **фазе 3** на реальной геометрии
   объектов.
